# Petria Editor

Онлайн-редактор BPMN-схем для бизнес-аналитиков.

## Структура репозитория

| Путь          | Что это                                                           |
| ------------- | ----------------------------------------------------------------- |
| `index.html`  | **v0** — старый прототип на vanilla JS + bpmn-js (CDN)            |
| `engine.js`   | **v0** — экспериментальный симулятор BPMN (пока не используется)  |
| `app/`        | **v1** — React + Vite + TypeScript, активная разработка           |

`petria.io` сейчас деплоится со старой версии (`index.html` в корне).
Перевод на v1 — задача недели 12 по плану.

## v1: запуск локально

```bash
cd app
npm install
npm run dev
```

Откроется на `http://localhost:5173`. Подробности — в `app/README.md`.

## План разработки

12 недель, **итог I этапа** — работающий BPMN-редактор с системой
веток и мерджа, базовый ЛК, приватные проекты.

| Неделя | Что делаем                                          |
| -----: | --------------------------------------------------- |
| 1      | Каркас Vite+React+TS, перенос редактора, localStorage |
| 2      | 6 базовых элементов BPMN, подписи, удаление, стрелки |
| 3      | UUID для всех элементов, JSON-формат, экспорт/импорт |
| 4      | Структура веток в данных (Process → Branch → snapshot) |
| 5      | UI веток: список, переключение, published / черновик |
| 6      | Свободное редактирование черновиков                 |
| 7      | Diff между ветками по UUID                          |
| 8      | Мердж черновика в опубликованную                    |
| 9      | История мерджей, откат к версии                     |
| 10     | Сущность «Проект», обкатка на 5–10 реальных процессах |
| 11     | FastAPI + PostgreSQL, регистрация, JWT, миграция данных |
| 12     | Деплой petria.io, Telegram-канал, первые BA         |

### Архитектурные решения, принятые сразу

1. **UUID на каждый элемент** схемы — фундамент под правила/документы во II этапе.
2. **Структура БД сразу под ветки** — процесс = много веток, главная — `published`.
3. **Аудит-лог с первого дня** — каждое действие пишется отдельной таблицей.
4. **JSON как формат хранения** — экспорт в BPMN 2.0 XML отдельная функция.
5. **Слой `dataLayer`** в коде с первого дня, чтобы переезд с localStorage на API
   на 11 неделе не требовал переписывания UI.
